# 1) 前置確認

* 請先向「京哥」確認：**主機 400 的 RM Code 是否已完成設定並已進入生效排程**

  * 建議確認項目：客戶統編、欲更改的 RM Code、提交/異動日期（作為 T 日）

> **T 日定義：** 本次在系統完成「更改 RM」並送出成功的日期。

---

# 2) 操作步驟

**Step 1**：員工入口網站 → 應用系統專區 → 客戶關係管理系統
**Step 2**：於搜尋欄輸入「客戶統編」→ 點擊確認
**Step 3**：點擊客戶旁的 icon → 開啟「目標客戶業務推展名單」
**Step 4**：點擊「更改 RM」→ 輸入 RM Code → 儲存/送出

---

# 3) 生效規則與對外說明口徑

完成更改 RM Code 後，系統生效時點如下：

* **主機 400：T + 1 日生效**
* **Call Report：T + 2 日生效**

---

# 4) 常見情境與處理原則

* RM/ARM 可能因急需撰寫 Call Report，而希望在 **T + 1 日** 就能操作。
* 但依規則 **Call Report 需 T + 2 日才會反映**，因此：

  * **不要用「已改好」當作可立即寫 Call Report 的保證**
  * 對外回覆建議明確帶日期：

    * 「400 會在 *（T+1）* 生效；Call Report 會在 *（T+2）* 才會更新。」
