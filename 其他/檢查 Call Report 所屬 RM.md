# 1) Call Report 的用途與建立位置

* **RM 會在 CRM 客戶關係系統建立 Call Report**

  * 用途：**記錄 RM 與客戶的往來/互動情形**（拜訪、聯繫、關懷等）

# 2) 「所屬」欄位 × CRM Call Report 的判讀規則

### A. 「所屬」欄位有 RM

* 意義：
  * **已建立 RM Code**
  * **主機 400 也有記錄**

* 補充查詢方式：
  * 可透過 Access 的 **「0 - 查詢客戶及RM資訊(精簡+)」** 查詢
  * 該查詢資料範圍：**都是已經有 RM Code 的客戶**

* Deleted 欄位解讀：
  * **Deleted = N**：表示**現有往來**
  * **Deleted = Y**：表示**已清除無往來**

### B. 「所屬」欄位沒有 RM，但 CRM 系統上有該客戶的 Call Report

* 意義：
  * **開放認養**（代表 CRM 已有人記錄互動，但尚未在「所屬」建立 RM Code/歸屬）

### C. CRM 系統上沒有該客戶（查不到 Call Report）
* 意義：
  * **尚未有人建立 Call Report**
  * 因此也可視為：**目前沒有互動紀錄被建立在 CRM 中**
